## ğŸ§  What are headers in a requests call?
Headers are key-value pairs sent in an HTTP request (or response) to provide meta-information about the request or the client.

Think of headers as:

* ğŸ‘¤ Identity cards (who's making the request?)

* ğŸ“¦ Format hints (what kind of data is being sent/received?)

* ğŸ” Security tokens (is the request authorized?)

## ğŸ”§ Common Use Cases for Headers
### 1. Content-Type
Tells the server what type of data you are sending.

```

    'Content-Type': 'application/json'
}
```

### 2. Authorization
Used for APIs that require authentication (e.g., Bearer tokens, Basic Auth).

```
headers = {
    'Authorization': 'Bearer <your_token>'
}
```

### 3. User-Agent
Identifies the client (browser, tool, etc.).

```
headers = {
    'User-Agent': 'my-awesome-script/1.0'
}
```
ğŸ§ª Example
```

    'Authorization': 'Bearer my_secret_token',
    'Content-Type': 'application/json',
    'User-Agent': 'kalyan-devops-script/1.0'
}

print(response.status_code)
print(response.json())

### ğŸ› ï¸ Using Parameters in `requests.get()`

Instead of manually building the query string, use the `params` argument:

```python

print("Data:", response.json())

```

## âœ… HTTP Methods Summary Table

| Method  | Purpose                          | Request Type | Body Support | Example Use Case                                  |
|---------|----------------------------------|---------------|--------------|---------------------------------------------------|
| `GET`   | Retrieve data from the server    | Safe/Idempotent | âŒ No       | Fetch user info, list resources                   |
| `POST`  | Send data to create a resource   | Non-idempotent | âœ… Yes      | Create a new user, submit a form                  |

### âœ… Notes:

- **Safe**: Does not change server state (`GET`, `HEAD`, `OPTIONS`)
- **Idempotent**: Repeating the same request yields the same result (`GET`, `PUT`, `DELETE`)
- **Non-idempotent**: Repeating may create multiple resources or have different effects (`POST`, `PATCH`)

## ğŸ–¼ï¸ Sending Files or Images with Python `requests`

You can send any file (text, image, etc.) using the `files` parameter in a POST request.

---

### ğŸ“‚ 1. Upload a File (e.g., `.txt` or `.log`)

```python
import requests

url = "https://httpbin.org/post"
file_path = "sample.txt"

with open(file_path, 'rb') as f:
    files = {'file': f}
    response = requests.post(url, files=files)

print(response.status_code)
print(response.json())
```

### ğŸ–¼ï¸ 2. Upload an Image (e.g., .jpg, .png)

```python

    files = {'image': ('image.png', img, 'image/png')}
    response = requests.post(url, files=files)

```

## â±ï¸ Setting Timeouts in Python `requests`

- **Connection timeout** â€“ time to establish a connection
